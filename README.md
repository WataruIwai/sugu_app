# Sugu

Sugu は、
「すぐ検索、すぐ理解、すぐ保存」
をコンセプトにした英単語学習アプリです。

複数のアプリを行き来せず、
単語検索から保存までを1つのアプリで完結できる体験を目指して開発しました。    
<br>
<br>
## なぜ作ろうと思ったのか

実際に自分が TOEIC の勉強をしていた時、
電車での移動時間を使って、
英語字幕で映画や動画を見ることが多くありました。

その中で、
わからない単語が出てきた時は、

- 単語をコピーする
- Google で意味を調べる
- 辞書サイトを開く
- 後で復習するためにメモアプリへ貼り付ける

という流れを毎回繰り返していました。

ただ、
この一連の流れでは、
スマホ内で何度もアプリやサイトを行き来する必要がありました。

特に電車移動中は、

- 次の駅まで数分しかない
- すぐ降りないといけない
- 時間が足りず調べるのを諦める
- メモだけ増えて復習しなくなる

ということが本当に多かったです。

「せっかく出会えた単語を、
もっと気軽に記録して、
もっと素早く覚えられたらいいのに」

そう思ったのが、
Sugu を作ろうと思ったきっかけでした。

単語を記録するアプリ自体は既にたくさんあります。

ただ、

- 保存までの操作が多い
- 検索と記録が分断されている
- アプリ単体で完結しない
- UI が複雑で使いづらい

と感じることが多く、
自分が本当に使いたいと思える体験を作りたいと思いました。

## 工夫したこと

### OpenAI API のトークン消費を抑える構成

OpenAI API を利用するにあたり、
APIトークン消費量をできる限り抑えることを意識しました。

単語検索時は、
まずDB内に既に検索済みデータが存在するか確認し、
存在する場合は OpenAI API を呼び出さず、
保存済みの意味データを返す構成にしています。

これにより、
同じ単語が再検索された場合でも、
不要な OpenAI API 呼び出しを避けられるようにしています。

---

### Sign in with Apple を採用したこと

認証周りについても、
途中で設計を見直しました。

当初はメールアドレス + パスワード認証を想定していましたが、

- ユーザー登録のハードルを下げたい
- パスワード忘れ対応の運用コストを減らしたい

と考え、
最終的に Sign in with Apple を採用しました。

これにより、
ユーザーは数タップで登録でき、
パスワード管理やリセット対応を行わずに利用できる構成にしています。
